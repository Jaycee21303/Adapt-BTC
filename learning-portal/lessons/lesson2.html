How Bitcoin Works

Bitcoin coordinates participants through a set of rules enforced by nodes, miners, and cryptography.

Blockchain

A chain of blocks that record transactions. Each block references the previous one, creating an immutable history.

Blocks

Blocks bundle transactions, metadata, and a proof-of-work solution. Nodes validate blocks against consensus rules.

Hashing

SHA-256 hashing secures blocks and addresses. Hash functions create fingerprints that are infeasible to reverse.

UTXOs

Unspent transaction outputs track spendable balances. Wallets build new transactions by consuming and creating UTXOs.

Proof of Work

Miners expend energy to find a valid hash, making it costly to rewrite history and aligning incentives.

Nodes

Full nodes enforce rules, relay transactions, and reject invalid blocks. Users can self-verify with their own node.

Finality

Each additional confirmation makes a transaction exponentially harder to reverse. Six confirmations are standard for large transfers.

Summary

  • Blocks link via hashes to form the blockchain.
  • Proof of Work makes altering history prohibitively expensive.
  • UTXOs define spendable balances and keep Bitcoin auditable.
